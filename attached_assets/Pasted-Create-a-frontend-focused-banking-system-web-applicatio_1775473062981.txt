Create a frontend-focused banking system web application optimized for Replit.

Important rules:
- All source code must be written in English.
- All user-facing interface text must be written in Brazilian Portuguese.
- The frontend must be the main focus of the project.
- The application must consume an existing REST API.
- The system must be designed to interact with a banking API that supports Individual Clients, Business Clients, withdrawals, statements, settings, and exports.
- Default interface language must be Portuguese.
- The settings screen must allow Portuguese and English selection.
- Currency and date format must be configurable in the frontend.
- The project must be easy to run in Replit.

# Main Goal
Build a complete frontend application for a banking system that interacts with an API.

The frontend must allow users to:
- visualize dashboard information
- create clients
- list clients
- filter clients
- paginate client lists
- edit clients
- delete clients
- perform withdrawals
- view statements
- change settings
- export data through API endpoints
- switch language and formatting options

# Frontend Stack
Use a frontend stack that works well in Replit.

Preferred stack:
- React
- TypeScript
- Vite
- React Router
- Axios or Fetch API
- React Hook Form
- Zod
- TanStack Query (optional but recommended)
- Tailwind CSS for styling
- shadcn/ui if helpful
- date-fns for date formatting

Keep the project practical and not overly complex.

# Application Type
This is a frontend-only project or frontend-first project.
Assume the backend API already exists.

The frontend must communicate with the API using HTTP requests.

# API Integration Assumptions
Assume the backend exposes endpoints such as:

## Individual Clients
- GET /api/individual-clients
- GET /api/individual-clients/:id
- POST /api/individual-clients
- PUT /api/individual-clients/:id
- DELETE /api/individual-clients/:id
- POST /api/individual-clients/:id/withdraw
- GET /api/individual-clients/:id/statement

## Business Clients
- GET /api/business-clients
- GET /api/business-clients/:id
- POST /api/business-clients
- PUT /api/business-clients/:id
- DELETE /api/business-clients/:id
- POST /api/business-clients/:id/withdraw
- GET /api/business-clients/:id/statement

## Settings
- GET /api/settings
- PUT /api/settings

## Exports
- GET /api/exports/individual-clients/csv
- GET /api/exports/individual-clients/pdf
- GET /api/exports/business-clients/csv
- GET /api/exports/business-clients/pdf

If needed, create a centralized API service layer to abstract these requests.
