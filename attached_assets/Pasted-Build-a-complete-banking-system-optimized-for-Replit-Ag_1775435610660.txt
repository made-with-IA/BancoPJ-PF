Build a complete banking system optimized for Replit Agent.

This is a full-stack web application with server-rendered pages and SQLite persistence.

Critical rules:
- All source code must be written in English.
- All user-facing interface text must be written in Brazilian Portuguese.
- Default interface language must be Portuguese.
- Settings must allow switching between Portuguese and English.
- Settings must also allow configuring currency format and date format.
- The project must be easy to run inside Replit with minimal setup.
- Use a simple, maintainable architecture. Do not overengineer.

# Main Objective
Create a banking system with support for Individual Clients and Business Clients, following MVC architecture, SQLite integration, business rules for withdrawals, unit tests for controllers, pagination, filters, edit/delete features, export to PDF and CSV, and a Portuguese-first interface.

# Technical Stack
Use:
- Node.js
- TypeScript
- Express
- SQLite
- EJS for server-rendered views
- Jest
- Supertest
- PDFKit
- csv-writer or equivalent
- Zod or class-validator for validation

Avoid unnecessary complexity.
Do not use a separate SPA frontend unless absolutely necessary.
Prefer a monolithic Express + EJS application because it is more stable for Replit.

# Business Rules
Implement the following mandatory rules:

1. Withdrawal must validate:
   - amount must be greater than zero
   - client must exist
   - sufficient balance is required
   - withdrawal amount must not exceed the maximum allowed for that client type

2. Withdrawal limits must be different:
   - Individual Client maximum withdrawal must be lower than Business Client maximum withdrawal

Use configurable constants such as:
- Individual Client max withdrawal: 1000
- Business Client max withdrawal: 5000

3. Every successful withdrawal must create a transaction record.

4. Statement must display:
   - transaction history
   - current balance
   - dates formatted according to settings
   - currency formatted according to settings

5. User-facing messages must be in Portuguese by default.

6. Language, currency, and date settings must affect rendered pages and exported reports.
